# 배경
현재 크롤링이 잘 되는지에 대한 테스트 파일이 없음

# 문제
1. 현재 크롤링이 잘 되는지에 대한 테스트를 할 수 있는 방법이 따로 존재하지 않음
2. 그렇기에 크롤링이 잘 되는지 보려면 어쩔수 없이 크롤링 까지 하는 로직을 실행해야함
3. 이러면 너무 비효율적임

# 요구 사항
1. source의 is_active에 맞게끔 활성화 된 url에 대해서 크롤링만 진행하는 로직 구현
2. 즉, 크롤링만 진행하여 게시글, 해당 게시글의 댓글만 크롤링을 진행하고 해당 게시물에 대한 요약을 진행하지 않도록 되어야함
3. 해당 로직은 실제 파일의 함수를 호출하여 진행되어야함. 별도의 로직을 또 구현하면 테스트하는 의미가 없기때문.
4. 해당 크롤링 진행 후 db의 item에 저장하는건 똑같이 수행함
5. 이는 별도의 파일(`test_crawl_only.py`)로 저장함.
