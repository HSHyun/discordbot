
현재 codex를 이용해서 요약하는 기능이 구현되어있고, 이를 테스트 하고 싶은데 현재 해당 코드를 따로 실행할 수 있는 부분이 구현되어있지 않음. 그러면 한번에 모든 코드를 처음부터 끝가지 실행해야하는데
이러면 문제가 코덱스가 요약을 해주는 부분에서 게시글이 단순히 모두 텍스트로 되어있으면 괜찮지만 이미지가 들어간 상태로 요약을 하면 너무 오래 걸림. 디시인사이드 50개의 게시물을 요약하는데 1시간정도의 시간이 걸림
그래서 한개의 게시글만 가져오고 그 게시물에 해당되는 것만 요약을 진행하는 코드가 필요

# 문제
1. 현재 테스트를 진행하는 별도의 파일이 존재하지 않음
2. 이러면 처음부터 끝가지 모든 로직을 실행해야하는데 codex가 이미지를 가진채로 요약하는게 너무오래걸림
3. 그래서 많은 게시물을 다 요약하는걸 기다리는건 사실상 너무 비효율적임

# 요구사항
1. 디시인사이드, 레딧 각각 별도의 파일로 구분하여 한개의 게시물에 대해서만 크롤링(api요청) 후 codex를 이용한 요약을 진행하는 코드 작성 → `test_dcinside_summary.py`, `test_reddit_summary.py`
2. 이는 코드를 구현하는게 아니라 원래 로직에서 사용되는 함수만 호출해서 작동하게끔 구현. test에서 따로 코드를 더 작성하면 test를 하는 의미가 없어지기 때문 → 기존 모듈 함수 재사용
3. 레딧의 경우에는 url를 고치면 바로 다른 url에 대해서도 똑같이 작동하게끔 구현 → 스크립트의 `--url` 인자로 즉시 다른 게시물 요약 가능
4. codex의 input과 output을 콘솔 출력했으면 좋겠음 → 두 스크립트 모두 입력·출력을 터미널에 표시
5. 한개의 게시물을 가져오는건 default값은 가장 최신 글이지만, 사용자가 추가로 url을 지정하면 해당 url로 크롤링 진행 → 기본은 최신 글, `--url`로 오버라이드 가능
6. 필요 시 다운로드한 이미지를 보존할 수 있도록 `--max-images`, `--keep-assets` 옵션을 제공
